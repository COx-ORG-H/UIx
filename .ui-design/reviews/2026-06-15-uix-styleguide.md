## Major Issues

### Issue 1: Sortable table headers were not keyboard-operable — FIXED
**Severity:** Major · **Category:** Usability/a11y · **Location:** `guide/app.js` (initTable)
**Problem:** `th[data-sort]` had a click handler but no `tabindex`/role/key handling — unreachable by keyboard.
**Fix applied:** `tabIndex=0` + `role="button"` + Enter/Space handler calling the same sort. `aria-sort` already toggled.

## Minor Issues

### Issue 2: Rich-select / menu / cmdk options lack arrow-key navigation
**Severity:** Minor · **Category:** a11y · **Location:** `app.js` setupRichSelect, menu/cmdk markup
**Problem:** Options are clickable and the popover focuses the search, but ↑/↓ + Enter to move/activate
options isn't wired (APG combobox/menu pattern). Mouse + type-to-filter work; pure-keyboard selection is limited.
**Recommendation:** Add roving `aria-activedescendant` + ↑/↓/Enter handling to the listbox. (Follow-up — larger.)

### Issue 3: CSS tooltip uses a `::after` pseudo-element
**Severity:** Minor · **Category:** a11y · **Location:** `components/tooltip.css`
**Problem:** `[data-uix-tip]::after` content isn't reliably announced by screen readers.
**Recommendation:** For real tooltips, pair with a visually-hidden element + `aria-describedby`, or use the
Popover API anchored tooltip. The CSS version is fine for purely decorative hints.

### Issue 4: KPI trend semantics in the demo
**Severity:** Minor · **Category:** Visual/content · **Location:** `index.html` #data-display KPI tiles
**Problem:** "Open tickets ↓12%" is shown with the down/red treatment, though fewer open tickets is *good*.
Demo content only — components are correct; tighten the sample data if used as a reference.

## Suggestions

### Suggestion 1: Concatenate CSS for production
`styles/main.css` pulls ~45 component files via `@import` (great for "copy only what you need", but a
render-blocking request chain). For a deployed build, concatenate/minify (or add the optional Style
Dictionary/bundling step noted in the README). Build-free local dev stays as-is.

## Positive Observations

- **Motion follows Emil's framework**: `--uix-ease-out` is his exact curve; UI durations <300ms; ⌘K opens
  instantly (keyboard/high-frequency); overlay **exits are faster than enters**; reduced-motion guard present.
- **No `transition: all`** anywhere — every transition names its properties.
- **WCAG AA in both themes** for every token pair, verified by an in-page contrast checker.
- **Not color-only**: status pills carry a dot **and** text; charts have legends; validation uses color + icon + text.
- **Native platform primitives**: `<dialog>` (focus trap + Esc), Popover API, `@starting-style` — animation without JS.
- **`<main>` landmark + `focus-visible`** rings throughout; sidebar favorites/collapse and table sort/filter/pin/column-visibility persist.
- **Form microinteractions** match the UXPin checklist: validate-on-blur (not per-keystroke), success "Looks good ✓",
  submit-button morph, helper/error text with icons.
